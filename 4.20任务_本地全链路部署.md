# 4.20 本地全链路部署任务书

> 目标：在本地 GPU 服务器上部署所有模型依赖，实现 VoiceAgentRAG 工程的完整运行（零外部 API 依赖）。

---

## 一、需要部署的模型总览

| # | 组件角色 | 模型名称 | 部署框架 | 用途 | 显存占用 |
|---|---------|---------|---------|------|---------|
| 1 | **LLM（大语言模型）** | Qwen2.5-7B-Instruct | **Ollama** | 对话生成、意图分类、ReAct 推理、记忆压缩 | ~6 GB |
| 2 | **Embedding（向量编码）** | bge-small-zh-v1.5 | **Ollama** | RAG 文档检索、语义缓存相似度计算 | ~0.5 GB |
| 3 | **STT（语音识别）** | SenseVoice-Small | **FunASR** | 语音转文字 + 情绪标签提取 | ~2 GB |
| 4 | **TTS（语音合成）** | CosyVoice2-0.5B | **CosyVoice2** | 文字转语音 + 音色克隆 | ~4 GB |

**总显存需求：约 12-13 GB**（推荐 RTX 4090 24GB / RTX 3090 24GB / A10 24GB）

---

## 二、逐组件部署步骤

### 2.1 LLM — Ollama + Qwen2.5-7B

Ollama 是最简单的本地 LLM 部署方式，一行命令即可启动 OpenAI 兼容 API。

```bash
# 1. 安装 Ollama（macOS / Linux）
curl -fsSL https://ollama.ai/install.sh | sh

# 2. 拉取模型（首次需下载 ~4.7GB）
ollama pull qwen2.5:7b

# 3. 验证模型可用
ollama run qwen2.5:7b "你好"

# 4. Ollama 默认以 daemon 运行，API 地址为 http://localhost:11434
# 验证 API
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen2.5:7b", "messages": [{"role": "user", "content": "你好"}]}'
```

**代码对接**：本项目已内置 `OllamaProvider`（`voice_optimized_rag/llm/ollama_provider.py`），通过 httpx 调用 Ollama API。

**性能参考**：
- 首 token 延迟（TTFT）：~200ms（GPU）/ ~2s（CPU）
- 生成速度：~40 tokens/s（GPU）/ ~5 tokens/s（CPU）
- 单次意图分类：~300ms → 比硅基流动 API 快 5-10 倍

**备选模型**（如果显存不够或需要更强能力）：
- `qwen2.5:3b` — 更小，~2GB 显存，速度更快，但能力下降
- `qwen2.5:14b` — 更强，~10GB 显存，推荐 24GB 显卡
- `deepseek-v2.5:7b` — 替代选项

### 2.2 Embedding — Ollama + bge-small-zh-v1.5

Embedding 模型用于将文本转为向量，供 FAISS 检索和语义缓存使用。

```bash
# Ollama 同时支持 embedding 模型
ollama pull nomic-embed-text
# 或使用中文优化的 BGE 模型（需通过 Modelfile 导入）

# 验证
curl http://localhost:11434/api/embed \
  -d '{"model": "nomic-embed-text", "input": ["测试文本"]}'
```

**代码对接**：本项目已内置 `OllamaEmbedding`（`voice_optimized_rag/retrieval/embeddings.py`），通过 Ollama 的 `/api/embed` 端点调用。

**配置项**：
```env
VOR_EMBEDDING_PROVIDER=ollama
VOR_EMBEDDING_MODEL=nomic-embed-text
VOR_EMBEDDING_DIMENSION=768
```

**备选方案**：如果需要更好的中文效果：
- 通过 `sentence-transformers` 加载 `BAAI/bge-small-zh-v1.5`（384 维，~100MB）
- 需要在 `embeddings.py` 中新增一个 `LocalEmbedding` 类（当前未实现，代码留有 `"local"` 接口位）

### 2.3 STT — FunASR + SenseVoice-Small

SenseVoice 是阿里达摩院的语音识别模型，单次推理同时输出**转写文字 + 情绪标签 + 事件标签**。

```bash
# 1. 安装 FunASR 框架
pip install funasr modelscope

# 2. 安装 PyTorch（需要与 CUDA 版本匹配）
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
# 如果是 CUDA 11.8：
# pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu118

# 3. 首次运行时 FunASR 自动从 ModelScope 下载模型（~500MB）
# 验证安装
python -c "
from funasr import AutoModel
model = AutoModel(model='iic/SenseVoiceSmall', trust_remote_code=True, device='cuda:0')
print('SenseVoice loaded successfully')
"
```

**代码对接**：本项目中 `voice_optimized_rag/voice/sensevoice_stt.py` 已封装完成，包括：
- `SenseVoiceSTT.transcribe()` — 离线整段推理
- `SenseVoiceSTT.transcribe_with_emotion()` — 推理 + 情绪提取
- `StreamingSenseVoiceSTT` — 基于 VAD + 分段的伪流式识别

**性能参考**：
- 推理速度：RTF ~0.05（即 1 秒音频只需 50ms 处理），比 Whisper large 快 15 倍
- 支持语言：中/英/日/韩/粤

**注意事项**：
- SenseVoice 本身是离线模型（需要完整音频段）
- 流式能力通过 `StreamingSenseVoiceSTT` 的 VAD + chunk 缓冲实现
- 首次加载模型约需 10-30 秒（后续已缓存）

### 2.4 TTS — CosyVoice2-0.5B

CosyVoice 是阿里达摩院的语音合成模型，支持标准合成、音色克隆、风格指令控制。

```bash
# 1. 克隆 CosyVoice2 仓库
git clone https://github.com/FunAudioLLM/CosyVoice2.git
cd CosyVoice2

# 2. 安装依赖
pip install -r requirements.txt

# 3. 模型会在首次运行时自动从 ModelScope 下载（~1.5GB）
# 验证安装
python -c "
from cosyvoice import CosyVoice2
model = CosyVoice2('iic/CosyVoice2-0.5B', load_jit=False, load_trt=False)
print('CosyVoice loaded successfully')
print('Available speakers:', model.list_available_spks())
"
```

**代码对接**：本项目中 `voice_optimized_rag/voice/cosyvoice.py` 已封装完成，包括：
- `CosyVoiceTTS.synthesize()` — 标准/克隆/指令三种模式
- `CosyVoiceTTS.synthesize_stream()` — 流式合成（asyncio.Queue 桥接）

**性能参考**：
- RTF ~0.3（1 秒语音需要 300ms 生成）
- 支持零样本音色克隆（传入 3-10 秒参考音频）

**注意事项**：
- CosyVoice2 需要 `cosyvoice` 包在 Python Path 中（pip install -e . 或 PYTHONPATH）
- 如果不需要音色克隆，也可用免费的 `edge-tts` 替代（无需 GPU）：
  ```env
  VOR_TTS_PROVIDER=edge
  ```

## 三、完整 .env 配置（全本地模式）

```env
# ─── LLM（本地 Ollama）───
VOR_LLM_PROVIDER=ollama
VOR_LLM_MODEL=qwen2.5:7b
VOR_LLM_BASE_URL=http://localhost:11434
VOR_LLM_TEMPERATURE=0.3
VOR_LLM_API_KEY=                      # Ollama 不需要 Key

# ─── Embedding（本地 Ollama）───
VOR_EMBEDDING_PROVIDER=ollama
VOR_EMBEDDING_MODEL=nomic-embed-text
VOR_EMBEDDING_DIMENSION=768

# ─── STT（本地 SenseVoice）───
VOR_STT_PROVIDER=sensevoice
VOR_SENSEVOICE_MODEL=iic/SenseVoiceSmall
VOR_SENSEVOICE_DEVICE=cuda:0          # 无 GPU 改为 cpu

# ─── TTS（本地 CosyVoice）───
VOR_TTS_PROVIDER=cosyvoice
VOR_COSYVOICE_MODEL=iic/CosyVoice2-0.5B
VOR_COSYVOICE_DEVICE=cuda:0           # 无 GPU 改为 cpu
VOR_COSYVOICE_DEFAULT_SPEAKER=中文女

# ─── 对话参数 ───
VOR_EMOTION_ANGRY_THRESHOLD=2
VOR_MEMORY_SHORT_TERM_TURNS=10
VOR_TRANSFER_MAX_FAILURES=3

# ─── Agent ───
VOR_AGENT_MAX_ITERATIONS=10
VOR_AGENT_TOOL_TIMEOUT=3.0

# ─── 向量数据库 ───
VOR_FAISS_INDEX_PATH=data/faiss_index

# ─── 日志 ───
VOR_SESSION_LOG_DIR=logs/sessions
```

## 四、启动顺序

```bash
# Step 1: 确认 Ollama daemon 在运行
ollama list    # 应看到 qwen2.5:7b 和 nomic-embed-text

# Step 2: 导入知识库（首次）

# Step 3: 启动 Web 服务
uvicorn web.server:app --host 0.0.0.0 --port 8000
```

## 六、无 GPU 降级方案

如果没有独立 GPU，可以用以下组合仍能跑通整个工程：

```env
VOR_LLM_PROVIDER=ollama          # CPU 推理（慢，但可用）
VOR_LLM_MODEL=qwen2.5:3b        # 更小的模型
VOR_STT_PROVIDER=none            # 跳过，用浏览器 Web Speech API
VOR_TTS_PROVIDER=edge            # 用微软 Edge TTS（免费，不需 GPU）
VOR_SENSEVOICE_DEVICE=cpu        # 如果仍想用 SenseVoice（慢 10x）
```

或者继续使用硅基流动 API 作为 LLM：
```env
VOR_LLM_PROVIDER=siliconflow
VOR_LLM_API_KEY=sk-你的Key
VOR_LLM_MODEL=deepseek-v3
VOR_STT_PROVIDER=none
VOR_TTS_PROVIDER=edge
```

## 七、代码中各组件与模型的对应关系

```
┌─────────────────────────────────────────────────────────────────┐
│  .env 配置                                                       │
│    ↓                                                             │
│  VORConfig (config.py)  ← Pydantic Settings 读取 .env           │
│    ↓                                                             │
│  create_llm(config)     → OllamaProvider   → Ollama API :11434  │
│  create_embedding(config) → OllamaEmbedding → Ollama API :11434 │
│  create_stt("sensevoice") → SenseVoiceSTT   → FunASR 本地推理    │
│  create_tts("cosyvoice")  → CosyVoiceTTS    → CosyVoice 本地推理 │
│    ↓                                                             │
│  Web Server (web/server.py)                                      │
│    lifespan() 初始化所有组件                                       │
│    /ws/chat    ← 文字/整段音频交互                                 │
│    /ws/stream  ← 流式音频交互                                      │
└─────────────────────────────────────────────────────────────────┘
```
