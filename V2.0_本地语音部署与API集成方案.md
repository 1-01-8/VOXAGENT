# V2.0 本地语音模块部署与 API 集成方案

## 一、当前状况

### 1.1 依赖问题回顾

首次运行测试时出现 `ModuleNotFoundError: No module named 'numpy'`，
原因是当前 Python 环境未安装项目依赖。通过以下命令一次性解决：

```bash
pip install -e ".[dev]"
```

该命令读取 `pyproject.toml` 中的 `[project.dependencies]` 和 `[project.optional-dependencies.dev]`，
安装了 numpy、faiss-cpu、pydantic、httpx、pytest、pytest-asyncio 等全部依赖。

> **建议**：项目根目录添加 `Makefile` 或 `scripts/setup.sh`，新人入项时执行一条命令即可。

### 1.2 测试基线

## 二、语音模块本地部署方案

### 2.1 硬件要求

| 模型 | 显存 | 推荐 GPU | 备选方案 |
|------|------|---------|---------|
| SenseVoice-Small (ASR) | ~2 GB | RTX 3060+ | CPU 可用(慢 5-10x) |
| CosyVoice2-0.5B (TTS) | ~3 GB | RTX 3060+ | CPU 可用(慢 10x+) |
| **合计** | **~5 GB** | RTX 3060 12GB | RTX 4090 最佳 |

> 两个模型可共用一张 GPU（错峰使用：ASR → 处理 → TTS 是串行流水线）。

### 2.2 部署架构

```
┌──────────────────────────────────────────────────────┐
│                   主应用 (Python)                      │
│  VORConfig(.env) ──→ SenseVoiceSTT / CosyVoiceTTS    │
│                      ↑ 直接调用模型                     │
│                      │                                │
│  [可选] 独立微服务模式:                                  │
│  FastAPI Server ──→ /v1/stt  (POST audio → text+emo)  │
│                 ──→ /v1/tts  (POST text → audio)      │
│                 ──→ /v1/tts/clone (POST text+ref)     │
│                 ──→ /v1/health                         │
└──────────────────────────────────────────────────────┘
```

### 2.3 两种集成模式

#### 模式 A：进程内直接调用（推荐开发/单机）

现有代码已支持。模型在首次 `synthesize()` / `transcribe()` 时自动加载（lazy init）。

```python
# 无需额外服务，直接使用
from voice_optimized_rag.voice.sensevoice_stt import SenseVoiceSTT
stt = SenseVoiceSTT(model_id="iic/SenseVoiceSmall", device="cuda:0")
result = await stt.transcribe(audio_bytes)
```

#### 模式 B：独立 FastAPI 微服务（推荐生产/多实例）

将模型包装为 HTTP API，主应用通过 HTTP 调用。优势：

- 模型进程独立，主应用崩溃不影响模型
- 可多实例部署、负载均衡
- 支持非 Python 客户端

**需要新建的文件：**

```
voice_optimized_rag/
  voice/
    api_server.py      ← FastAPI 服务入口
    api_stt_client.py   ← 替代直接调用的 HTTP 客户端
    api_tts_client.py   ← 替代直接调用的 HTTP 客户端
```

## 三、.env 配置方案

### 3.1 可以直接用 .env 吗？

**可以，而且已经原生支持。**

`VORConfig` 基于 `pydantic_settings.BaseSettings`，配置中已声明：

```python
model_config = {"env_prefix": "VOR_", "env_file": ".env", "extra": "ignore"}
```

所有配置项均可通过 `VOR_` 前缀的环境变量覆盖。

### 3.2 .env.example 字段说明

```bash
# ─── LLM ───
VOR_LLM_PROVIDER=openai          # openai / anthropic / ollama / gemini
VOR_LLM_MODEL=gpt-4o-mini
VOR_LLM_API_KEY=sk-xxx           # 必填（ollama 除外）
VOR_LLM_BASE_URL=                # ollama: http://localhost:11434/v1

# ─── Embedding ───
VOR_EMBEDDING_PROVIDER=openai    # openai / ollama / local
VOR_EMBEDDING_MODEL=text-embedding-3-small
VOR_EMBEDDING_DIMENSION=1536

# ─── Voice: ASR (SenseVoice) ───
VOR_STT_PROVIDER=sensevoice
VOR_SENSEVOICE_MODEL=iic/SenseVoiceSmall
VOR_SENSEVOICE_DEVICE=cuda:0     # cpu / cuda:0 / cuda:1

# ─── Voice: TTS (CosyVoice) ───
VOR_TTS_PROVIDER=cosyvoice
VOR_COSYVOICE_MODEL=iic/CosyVoice2-0.5B
VOR_COSYVOICE_DEVICE=cuda:0
VOR_COSYVOICE_DEFAULT_SPEAKER=中文女
VOR_COSYVOICE_REFERENCE_AUDIO=   # 声音克隆参考音频路径(可选)

# ─── Dialogue ───
VOR_EMOTION_ANGRY_THRESHOLD=2
VOR_MEMORY_SHORT_TERM_TURNS=10
VOR_TRANSFER_MAX_FAILURES=3

# ─── Agent ───
VOR_AGENT_MAX_ITERATIONS=10
VOR_AGENT_TOOL_TIMEOUT=3.0

# ─── Knowledge Base ───
VOR_KB_CONFIDENCE_THRESHOLD=0.75
VOR_FAISS_INDEX_PATH=data/faiss_index

# ─── Logging ───
VOR_SESSION_LOG_DIR=logs/sessions
```

### 3.3 配置优先级

```
代码默认值 < .env 文件 < 系统环境变量 < 构造函数参数
```

## 四、下一步实施计划

### Phase 1：模型下载与验证

```bash

from voice_optimized_rag.voice.sensevoice_stt import SenseVoiceSTT

"
```

```bash

# 2. 填入 API Key

```

### Phase 3：（可选）微服务 API 封装

如果需要将语音模型作为独立服务暴露：

| 端点 | 方法 | 输入 | 输出 |
|------|------|------|------|
| `/v1/stt` | POST | `audio: bytes` | `{text, emotion, language}` |
| `/v1/tts` | POST | `{text, speaker, mode}` | `audio/wav` stream |
| `/v1/tts/clone` | POST | `{text, ref_audio}` | `audio/wav` stream |
| `/v1/health` | GET | — | `{status, gpu_memory}` |

| 指标 | 目标 | 测量方法 |

| TTS 首音延迟 | < 300ms | 发送短文本，测量首个音频块 |
| E2E 往返延迟 | < 2s | ASR + LLM + TTS 全链路 |

## 五、风险与缓解

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| GPU 显存不足 | 模型加载失败 | 切换 CPU 模式 / 量化模型 |
| CosyVoice 依赖冲突 | 安装失败 | 使用 Docker 隔离 / conda 虚拟环境 |
| SenseVoice 流式延迟 | 首字慢 | 先用 VAD 分段，批量送入 |
| funasr 版本更新 | API 不兼容 | 锁定版本至 pyproject.toml |
